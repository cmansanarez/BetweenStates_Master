# Between States — Enhancement Specification
### Bridging Pitch Site + XR Prototype

This document outlines required enhancements to align the Between States project with final evaluation criteria.

The goal is to:
1. Strengthen the conceptual pitch (investor + academic audience)
2. Improve clarity of XR-specific design and system architecture
3. Integrate proof of working prototype into the experience
4. Prepare the project for further development

---

## 1. PERSONA SECTION (REQUIRED)

### Objective
Clearly define the target users with visual and structured persona cards.

### Requirements

Create a dedicated section: `# Personas`

### Primary Persona (70% emphasis)

Include:
- Name (e.g. “Alex, 26”)
- Context: Festival attendee
- Demographics:
  - Age: 18–35
  - Environment: concerts, raves, festivals
- Goals:
  - enhance live music experience
  - create unique content
  - feel immersed
- Behaviors:
  - records video
  - uses filters and social media
- Pain Points:
  - passive viewing through phone
  - lack of interactivity
- Quote (short, humanizing)

### Secondary Persona (30% emphasis)

- Role: Festival producer / queer arts organizer
- Goals:
  - increase engagement
  - differentiate event
- Needs:
  - scalable solution
  - customizable visuals

### Implementation (Site)
- Use card-based layout
- Include visual avatar or stylized imagery
- Use consistent color system from design palette

---

## 2. INFORMATION ARCHITECTURE (CRITICAL)

### Objective
Visualize how the system works across devices and environments.

### Requirements

Create section: `# System Architecture`

Include diagram showing:

- Participant smartphone
- Camera input (front + rear)
- Microphone input
- Live audio environment (DJ / stage)
- Web app (Hydra + p5)
- Visual output (AR overlay)
- QR code entry point
- Network layer (Wi-Fi / cellular)

### Implementation

- Use diagram (SVG, canvas, or styled HTML)
- Include directional arrows
- Label all nodes clearly

## 3. XR DEPTH + SPATIAL LAYERING

### Objective
Make it visually clear this is not a flat mobile app.

### Requirements

Create section: `# XR Experience Layers`

Show:

- Environment layer (rear camera)
- Identity layer (front camera face tracking)
- Visual layer (Hydra output)
- Interaction layer (audio + motion)

### Implementation

- Use exploded diagram or layered visual
- Consider parallax scrolling or depth effects
- Use gradients and transparency to show layering
